    Chisel

    Chisel is Quartz's Command Line Tool, Chisel can be used for a lot of uselfull features. And is easy extendable. Chisel plugs directly into the core, this means it has all he features available that are featured inside the core.

    Chisel is easy to edit, reuse or even remove. It has it's own folder and all it's dependancys are loaded via the main comopser file, just stripe these down and your ready to go.

      Stack

      This is the part where Quartz realy comes in. Quartz handles a stack, which is basicly a simple variable given in the root class. Everything that has something to do with data has to come trough the Stack. So you got an overly complicated variable that stores a shit load of useless variables and data? Well if you ask it like that.. Yes. But keep in mind that the Stack is extremely easy to write and read to / from the data inside is automaticy depandancy injected and even the slightest of your changes is easy to pull, since .. guess what yes ! it's in the stack! But this is a bad habbit? No and yes, keeping a way to big variable is not efficient at all, thats why Quartz manages your stack and it's uses. If there is non used content in your stack we won't use it. If you ahve a piece of content thats used a lot we'll give it priority. If the piority isn't good engough. We might actualy put it in memory for you!
